# 墨衡 · 桌面驾驶舱能力并入 APP 设计说明

**日期**: 2026-07-20  
**状态**: 待用户确认后实现原型  
**关联**: `docs/2026-07-14-墨衡-design.md`（原桌面三栏方案）

## 背景

设计文档中的「桌面驾驶舱」（左五维雷达 / 中目标看板 / 右提醒与复盘）从未在代码中落地。  
APP 端四 Tab（衡 / 习 / 复 / 我）已具备目标、习惯、复盘数据。本次将驾驶舱三栏能力按现有 Tab 拆分并入，先交付可点原型。

## 目标

在不新增第五 Tab 的前提下，让手机端获得与桌面驾驶舱等价的信息密度：

1. 看见五维态势（雷达 + 分数）
2. 看见目标进度看板
3. 看见按时间排序的今日提醒

## 决策摘要

| 项 | 选择 |
|----|------|
| 信息架构 | **方案 B**：拆进现有 Tab |
| 衡页处理 | **叠加**：保留今日衡盘 / 主攻环，其下插入新模块 |
| 实现路径 | **方案一**：衡页叠段 + 习页时间线，复用真实 state |
| 交付形态 | 先出可交互原型并推送线上 |

## 信息架构（改后）

### 衡

自上而下：

1. **保留**：问候文案、主攻维大卡 + 进度环、救火提示  
2. **新增「五维态势」**：五边形雷达图 + 五维分数列表（主攻维行高亮）  
3. **强化「目标看板」**：主攻 / 保底分区；卡片含描述、本周进度文案、横向进度条（对齐桌面看板视觉）  
4. **保留**：今日习惯条（可放在看板下方）

### 习

1. **新增「今日提醒」时间线**：按习惯 `remindAt` 升序；行内展示时间｜标题｜维度/类型｜完成状态  
2. **保留**：现有习惯列表与打卡 / 提醒开关

### 复

不变。保存五维打分后，衡页雷达与分数列表同步刷新。

### 我

不变。切换主攻维后，五维态势中的「主攻」高亮同步更新。

## 数据与渲染

| UI | 数据源 | 说明 |
|----|--------|------|
| 雷达 / 分数列表 | `state.scores` | 五维 1–10；无分时显示占位或 0 |
| 目标看板 | `state.goals` + 现有 `calcGoalProgress` | 主攻 / 保底分组与现有逻辑一致 |
| 提醒时间线 | `state.habits`（`remindAt`、`remindOn`、打卡状态） | 仅展示开启提醒的习惯；按时间排序 |
| 主攻高亮 | `state.focusDim` | 分数行与雷达轴标签标记「主攻」 |

空状态：无本地数据时注入轻量演示数据（目标 / 习惯 / 分数各若干条），便于评审一眼看懂；用户真实数据写入后优先用真实 state。

## 交互

- 目标卡片：点击进入现有编辑弹窗；打卡按钮行为不变  
- 时间线项：点击可跳转或滚动到习页对应习惯（原型阶段可仅展示，打卡仍在列表）  
- 雷达：只读；分数变更仅来自「复」页保存  

## 视觉约束

- 延续墨衡石灰底 + 墨字 + 砚青；避免新引入第三套色板  
- 雷达用 SVG，移动端单列全宽，高度约 200–240px  
- 时间线卡片白底圆角，左时间右内容，与桌面右栏节奏接近  
- 不破坏现有底部 Tab 贴底与 safe-area 规则  

## 非目标（本原型不做）

- 真正的桌面三栏 `.workspace` 布局  
- 连续 2 周 &lt;4 分的完整救火规则升级（沿用现有救火逻辑）  
- 新增独立「驾驶舱」Tab  
- 改动「复」「我」信息架构  

## 验收标准

1. 衡页在主攻环下方可见雷达 + 五维分数，主攻维高亮  
2. 衡页目标区呈现看板式进度条卡片，打卡后进度更新  
3. 习页顶部可见今日提醒时间线，时间排序正确  
4. 「复」保存分数后，回到衡页雷达与列表数字已更新  
5. 手机 Safari / 主屏幕 PWA 均可滚动浏览，底部 Tab 不被破坏
